# M3U Scanner 功能详解

## 🎯 已实现的所有功能

### 1️⃣ M3U 文件处理
- ✅ 解析本地 M3U/M3U8 文件
- ✅ 解析远程 M3U URL
- ✅ 支持 EXTINF 标签解析
- ✅ 解析频道属性（group-title, tvg-id, tvg-logo 等）
- ✅ 支持 EPG URL 提取

### 2️⃣ 扫描功能
- ✅ 并发扫描（可配置 1-100 线程）
- ✅ 快速检测模式（仅验证可用性）
- ✅ 完整检测模式（获取详细流信息）
- ✅ 可配置超时时间（5-120 秒）
- ✅ 实时进度显示
- ✅ 支持中途停止（已修复闪退问题）
- ✅ 自动重试机制

### 3️⃣ 流信息展示
**视频信息：**
- 编码器（H.264, H.265, VP9 等）
- Profile（High, Main, Baseline 等）
- 分辨率（宽x高）
- 帧率（fps）
- 视频码率（kbps）
- 像素格式（yuv420p 等）

**音频信息：**
- 编码器（AAC, MP3, AC3 等）
- 声道数和布局（2.0, 5.1 等）
- 采样率（Hz）
- 音频码率（kbps）
- 语言标签

**容器信息：**
- 格式名称
- 总码率
- 探测分数

### 4️⃣ 缩略图功能
- ✅ 自动截取视频第 2 秒画面
- ✅ 缩略图缓存机制
- ✅ 自动缩放到 320px 宽度
- ✅ Base64 编码传输
- ✅ 异步加载不阻塞界面

okhttp: okhttp/3.10.0

## 🔧 技术亮点

### FFmpeg/FFprobe 智能查找
优先级：
1. **程序同目录** - `./ffmpeg.exe`, `./ffprobe.exe`
2. **当前工作目录** - 运行时所在目录
3. **系统 PATH** - 环境变量配置

### 并发扫描引擎
- 使用 goroutine 池控制并发
- Channel 通信实时更新进度
- Context 控制优雅退出
- Mutex 保护共享数据（修复闪退问题）

### 缩略图生成
```bash
ffmpeg -i <stream_url> -ss 00:00:02 -vframes 1 -vf scale=320:-1 -q:v 5 output.jpg
```
- 跳到第 2 秒（避开黑屏）
- 截取 1 帧
- 缩放到 320px 宽度
- 质量级别 5

### 实时通信
- Server-Sent Events (SSE) 推送更新
- 无需轮询，节省资源
- 自动重连机制

## 📊 性能参数

| 配置 | 默认值 | 范围 | 说明 |
|------|--------|------|------|
| 并发数 | 20 | 1-100 | 过高可能被限流 |
| 超时 | 15秒 | 5-120秒 | 根据网络调整 |
| 快速模式 | 关闭 | - | 牺牲详细信息换速度 |

**扫描速度参考：**
- 快速模式：100个频道约 30-60 秒（并发20）
- 完整模式：100个频道约 2-5 分钟（并发20）

## 🚨 注意事项

1. **并发数不要过高** - 可能触发源服务器限流
2. **User-Agent 很重要** - 部分源需要特定 UA
3. **网络稳定性** - 建议稳定网络环境下使用
4. **FFmpeg 版本** - 建议使用最新版本

## 🎉 使用示例

```bash
# 1. 启动程序
./m3u-scanner.exe

# 2. 打开浏览器访问 http://localhost:8080

# 3. 加载示例文件
点击"打开文件" -> 选择 example.m3u

# 4. 开始扫描
点击"开始扫描"

# 5. 查看结果
点击任意频道查看详细信息和缩略图

# 6. 播放测试
点击"播放" -> 选择"在浏览器中播放"
```
